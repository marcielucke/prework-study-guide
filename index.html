HTML

  • the head section will contain information about the webpage for the computer
  • Elements in the head include various types of metadata and is not visible to the user.
  • Content in the head will include information such as the language being used, the title, the style, the URL, and more.
  • The body element represents the visible content shown to the user.
  • The page will consist of a header, main, and footer section of work
  • HTML is the bones of the webpage that gives foundation to the other tools
  • start this document with to notify the computer what tool we are using.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • CSS includes inline CSS, internal style sheets, and external style sheets
  • Inline CSS only applies to a specific line
  • Internal style sheets are CSS rules for an entire section of the HTML file
  • External CSS style sheets include inserting a link that applies all CSS rules from a separate form

Git

  • git status checks what branch we are in
  • git checkout -b branch-name creates a new branch and switches to it
  • git commit -m “(message)” Commits the changes with a custom message.
  • git workflow includes add, commit, & push work to your feature branch.
  • After this step you will open a pull request on github and merge to your main branch.

Javascript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • fuctions are a way of packaging functionality that you wish to reuse.
  • Javascript reads code from top to bottom. Place java file near end of HTML document so that it affects all of the script
  • If you see something that looks like a variable name that it is followed by (), it is likely a function
